UI设计稿那些事

发表于 2016-05-28   |   分类于 前端   |     |  

这里不扯UI怎么体现需求方的设计，这里就聊聊UI设计稿前端怎么切的那些事。

起因

有一天早上我来到公司，打开项目需求管理软件，多了一个需求。

看到没，” 2.0 “ 。整个博客重新设计和实现，不是所谓意义上的 2.0 升级。
话不多说，我拿到原型设计和设计稿就开始梳理起来；一部分一部分的列出来；头部、菜单、搜索框、内容列表、…、推荐、评论、底部；然后就看看那些可以弄成公用模块，组件化、需要什么插件，图片怎么处理，数据交互怎么个弄法啊，动画效果，还有兼容性，等等…

（过了一段时间….）

来看看我们网站的 images 文件夹吧。/(ㄒoㄒ)/~~

该合并的没合并，文件命名不规范，没有一套基础图标或font-awesome。

UI到前端桥梁

在不算很久之前，也是传统一点行业的时候，一般网页设计，就是所谓“美工”、“切图仔”；甚至还会写点代码，来点HTML和CSS，甚至来点Flash。想当年网页三剑客（Dreamweaver、Fireworks、Flash）可红遍天下了。
后来软件行业发展迅速，网站的体现和效果要求也越来越高，再加上硬件设备提升了几个档次。一切都来得那么及时。

而当用户看到的网站界面，不只是UI设计师呕心沥血创作的效果图，还有前端开发人员将设计稿一个个单独的切图经用技术实现出来。切图作为UI设计师与前端之间的桥梁，它的作用很关键，合适的切图、精准的位置可以最大限度的还原效果图的设计，精妙的切图更会有事半功倍的效果哦！

从之后网页设计演变到UI设计和前端开发两个分支。

切图

所谓手中无刀，心中有刀。
切图，是一切UI到前端的开始。

简单的切图就是整张图片分类的输出，其中有背景图也好，横幅也好，小图片也好。最后就是一些分类出来。而背景图的处理，透明图化或者平铺的处理方式，效果图处理。
再到图标处理，就是平时大家会说的icon、icon、icon …
“点九”图片也是。
甚至一些效果，比如阴影啊，统一按钮样式，1px边框的特殊处理。

大部分都是使用PS（photoshop）或者PS切图插件。
周边也有很多非常专注的工具。

  • Cutterman
  • Assistor PS
  • Cut&Slice me
  • DevRocket
  • Slicy

CSS精灵图

CSS Sprites其实就是把网页中一些背景图片整合拼合成一张图片中，再利用DIV CSS的“background-image”，“background- repeat”，“background-position”的组合进行背景定位，background-position可以用数字能精确的定位出背景图片在布局盒子对象位置。

CSS Sprites适用小图标icon类素材，一般图标很小十多像素几十像素的宽度高度；且很少局部更新。

优点

减少http iis请求数，提升了网站性能，同时也减少图片文件数目。

缺点

图片合并比较麻烦，要留出合理空间，即不出现干扰图片的情况，且又不能极大浪费空间；如需更新，尤其是改动，比较繁琐。由于图片的位置需要固定为某个绝对数值，这就失去了诸如center之类的灵活性。

小结（DIVCSS5推荐小结）

犹豫拼接图片需要一定经验技巧（做实践即可快速掌握）、测量定位数值、修改不是那么灵活等原因，一般小网站站、小流量网站、一般企业网站不是很推荐使用CSS Sprites，因为使用CSS Sprites会比普通单个背景图片布局要耗费时间和精力，所以不是很推荐小站。但这个布局技巧也必须要学会掌握灵活布局才是目的。小站HTTP请求数丰富这点拼接其实也起不来什么优势反而会浪费宝贵时间。相反大网站大流量网站推荐使用，这样比较值得。

font-awesome

一个大型网站小图标处处可见，对前端开发人员来说，对这样的小图标都需要像对图片一样来处理多麽繁琐。有人说不是可以用CSS Sprites吗？哦啊
可是如果颜色不只是黑白或者一种颜色呢，如果大小不一致呢；你会发现，前端人员会崩溃死的，将在这上面耗费大量资源和精力。

我们来看看这个玩意，它拥有将近600个常用icon图标（不断更新中），可使用CSS控制样式，包括大小、颜色、阴影等等，而且还开源，甚至完全免费，大公司都用。你说这玩意好不好 …

Font Awesome 是一款可缩放的矢量图标字库，你可以利用CSS将Font Awesome发挥到极致，包括：大小、颜色、阴影或者其它任何支持的效果。

它的缺点是：
1、对IE6等浏览器兼容性不是很好，不过最新版本的Font Awesome和IE7完美兼容。
2、要想用fontawesome，需要引入相关的字体文件，增加了网页大小。

可以关注一下阿里巴巴的iconfont，也是个不错的东西，可对比一下它和font-awesome的优缺点。

标注、测量

UI设计师将设计稿给到前端开发，最好做好标注和测量。前端才会更好更方便地处理，而且高保真地产出页面。

前端开发中会很好把握哪些样式要提炼出来，哪些要弄成组件。
